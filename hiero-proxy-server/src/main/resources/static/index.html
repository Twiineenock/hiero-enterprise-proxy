Quick start — one command

docker
docker run \
  -e HEDERA_ACCOUNT_ID=0.0.xxxxx \
  -e HEDERA_PRIVATE_KEY=your_private_key \
  -e HEDERA_NETWORK=hedera-testnet \
  -p 8080:8080 \
  ghcr.io/hiero-hackers/hiero-enterprise-proxy:latest

Call it from Python

python
import requests
BASE = "http://localhost:8080/api/v1"

# Create an account funded with 10 HBAR
account = requests.post(f"{BASE}/accounts", json={"initialBalanceInHbar": 10}).json()
print(account["accountId"])  # 0.0.12345

# Create a fungible token
token = requests.post(f"{BASE}/tokens", json={"name": "MyToken", "symbol": "MTK"}).json()
print(token["tokenId"])   # 0.0.55001

Call it from JavaScript

javascript
const BASE = "http://localhost:8080/api/v1";

// Create a topic and publish a message
const topic = await fetch(`${BASE}/topics`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ memo: "my event bus" })
}).then(r => r.json());

await fetch(`${BASE}/topics/${topic.topicId}/messages`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ message: "Hello Hiero" })
});

Why use this?

🐳

Docker-first

One command to run. No Java, no Maven, no SDK setup required on your machine.

🌐

Any language

Python, JavaScript, Go, PHP, Ruby — if it can make an HTTP request, it can talk to Hiero.

📖

Full OpenAPI spec

Auto-generate a typed client in any language from the spec at /v3/api-docs.

🔑

Server-side keys

Key generation and rotation happen on the server. You receive the key once — store it safely.

🛡️

One operator account

Configure once. All transaction fees flow through the operator — your clients stay clean.

⚡

Structured errors

All errors follow RFC 7807 — consistent, machine-readable, and easy to handle.
